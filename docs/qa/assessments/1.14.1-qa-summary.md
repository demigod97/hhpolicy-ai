## Top 5 Highest Risk Edge Cases

### 1. EC-001: Rapid Upload-Delete Race Condition (P0)

**Risk Profile:**
- **Probability:** Medium (users frequently delete after upload)
- **Impact:** High (orphaned N8N jobs, resource leaks)
- **Detectability:** Low (silent failure)

**Mitigation:**
- Edge function checks if document still exists before processing
- Mark document status before N8N call
- Idempotent processing

**Implementation Time:** 2 hours
**Test Code:** Ready in `1.14.1-edge-case-implementations.md`

---

### 2. EC-005: Malformed Embeddings Validation (P0)

**Risk Profile:**
- **Probability:** Low (depends on N8N stability)
- **Impact:** Critical (silently breaks RAG, incorrect AI responses)
- **Detectability:** Very Low (only user complaints)

**Mitigation:**
- Validate embedding dimensions (must be 1536)
- Validate all values are numeric
- Validate range (typically [-1, 1])
- Reject invalid embeddings, mark as failed

**Implementation Time:** 3 hours
**Test Code:** Complete validation function provided

---

### 3. EC-011: Signed URL Expiration During Processing (P0)

**Risk Profile:**
- **Probability:** High (will happen with large documents)
- **Impact:** High (processing fails silently)
- **Detectability:** Medium (users report processing stuck)

**Mitigation:**
- Configure signed URL TTL > max processing time (2 hours recommended)
- Monitor processing time vs. TTL
- Implement URL refresh mechanism if N8N reports 403

**Implementation Time:** 1 hour
**Test Code:** Configuration validation provided

---

### 4. EC-012: Unauthorized Callback Spoofing (P0)

**Risk Profile:**
- **Probability:** Low (requires attacker knowledge)
- **Impact:** Critical (data corruption, malicious embedding injection)
- **Detectability:** Low (appears as legitimate processing)

**Mitigation:**
- Validate N8N auth token on all callbacks
- Rate limit unauthorized attempts (max 10/minute)
- Log all unauthorized attempts
- Alert on excessive unauthorized attempts

**Implementation Time:** 2 hours
**Test Code:** Complete auth middleware provided

---

### 5. EC-020: No Extractable Text (Scanned PDFs) (P1)

**Risk Profile:**
- **Probability:** High (20-30% of document uploads)
- **Impact:** Medium (user confusion, wasted processing)
- **Detectability:** High (users notice immediately)

**Mitigation:**
- Detect scanned documents during processing
- Mark as "completed" with special flag (not failed)
- Display user-friendly message: "No text found - may be scanned images"
- Suggest OCR processing as future feature

**Implementation Time:** 2 hours
**Test Code:** Complete handler logic provided

## Production Monitoring & Alerting

### Critical Alerts (Set Up Immediately)

```sql
-- Alert 1: Race Condition Detector
-- Trigger: Documents deleted while processing
SELECT COUNT(*) as orphaned_jobs FROM sources
WHERE processing_status = 'processing'
AND deleted_at IS NOT NULL;
-- Alert if > 0

-- Alert 2: Orphaned Processing Jobs
-- Trigger: Documents stuck in 'processing' > 1 hour
SELECT COUNT(*) as stuck_jobs FROM sources
WHERE processing_status = 'processing'
AND updated_at < NOW() - INTERVAL '1 hour';
-- Alert if > 5

-- Alert 3: Invalid Embeddings
-- Trigger: Documents with wrong embedding dimensions
SELECT COUNT(*) as invalid_embeddings FROM documents
WHERE array_length(embedding, 1) != 1536;
-- Alert if > 0

-- Alert 4: Failed Document Backlog
-- Trigger: Too many documents stuck in 'failed' status
SELECT COUNT(*) as failed_backlog FROM sources
WHERE processing_status = 'failed'
AND updated_at > NOW() - INTERVAL '24 hours';
-- Alert if > 10

-- Alert 5: Processing Success Rate
-- Trigger: Success rate drops below 95%
SELECT
  COUNT(CASE WHEN processing_status = 'completed' THEN 1 END)::float /
  COUNT(*)::float as success_rate
FROM sources
WHERE created_at > NOW() - INTERVAL '1 hour';
-- Alert if < 0.95
```

## Test Infrastructure Requirements

### Mock N8N Server

**Purpose:** Fast integration testing without real N8N dependency

**Features:**
- Simulates N8N webhook behavior
- Records webhook calls for assertion
- Configurable delays (simulate slow processing)
- Error injection (simulate timeouts, failures)
- Automatic callback to Supabase

**Implementation:**
```typescript
// Simple Express server
import express from 'express';

const app = express();
const webhookCalls: any[] = [];

app.post('/webhook/process-document', async (req, res) => {
  webhookCalls.push(req.body);

  // Simulate processing delay
  await new Promise(resolve => setTimeout(resolve, 1000));

  // Call back to Supabase
  await fetch(`${req.body.callback_url}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      source_id: req.body.source_id,
      extracted_text: 'Mock extracted text',
      chunks: [
        { text: 'Mock chunk', embedding: Array(1536).fill(0.5), chunk_index: 0 }
      ]
    })
  });

  res.json({ success: true });
});

app.listen(5678, () => console.log('Mock N8N listening on :5678'));
```
