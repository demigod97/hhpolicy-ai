## 🛠️ Technical Details

### Edge Function Flow

**Upload → Process → Callback:**

```
1. User uploads PDF via DocumentUploader
   ↓
2. useFileUpload.uploadFileAndProcess()
   - Creates source record (status: 'pending')
   - Uploads file to storage
   - Updates status to 'processing'
   - Invokes edge function: process-document
   ↓
3. process-document edge function
   - Creates signed URL (1 hour expiry)
   - Calls N8N webhook with:
     * source_id
     * signed file_url
     * callback_url
   ↓
4. N8N workflow processes document
   - Extracts text
   - Generates embeddings
   - Stores in vector database
   ↓
5. N8N calls process-document-callback
   - Updates source status: 'completed'
   - Stores extracted content
   ↓
6. Real-time subscription updates UI
   - Document shows "Completed" status
   - Available for chat
```

### Error Handling

**Edge Function Errors** → Status set to 'failed'
**N8N Timeout** → Status remains 'processing' (no callback)
**Callback Errors** → Status remains 'processing'

**Manual Recovery**: Re-trigger edge function manually if needed

## 🎓 What I Learned

### Supabase Best Practices
- Modern edge functions use `Deno.serve` not `serve`
- Use `npm:` prefix for imports, not `esm.sh` or `deno.land/x`
- Signed URLs have expiry (default 1 hour)
- Auto-populated secrets: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`

### Development Workflow
- CLI tools may fail in certain environments → always have manual backup
- Port auto-selection can confuse users → document or hardcode
- Edge function testing requires logs monitoring
- N8N integration requires correlation between source_id and callbacks
