# Archived Project Management Documents

This directory contains historical project management documents including deployment logs, testing guides, and implementation summaries from previous development sessions.

**Last Updated**: 2025-10-20

---

## 📁 Directory Structure

```
archived/
├── deployments/        # Historical deployment logs and guides
├── testing/           # Testing guides and phase documentation
└── implementations/   # Implementation summaries and completion reports
```

---

## 📂 Subdirectories

### deployments/
Contains logs and guides from production deployments.

**Naming Convention**: `YYYY-MM-DD-deployment-description.md`

**Contents**:
- Day 1 deployment guides
- Migration deployment logs
- CopilotKit deployment attempts
- Edge function deployment records

**Usage**: Reference for understanding deployment history and troubleshooting deployment issues.

---

### testing/
Contains testing phase documentation and guides.

**Naming Convention**: `phase-X-testing.md` or `YYYY-MM-DD-testing-description.md`

**Contents**:
- Phase 1 testing guides
- Phase 2 testing documentation
- Verification guides
- Step-by-step testing instructions

**Usage**: Historical testing approaches and methodologies.

---

### implementations/
Contains implementation summaries and completion reports from development sessions.

**Naming Convention**: `epic-X.Y-description.md` or `feature-implementation-summary.md`

**Contents**:
- Epic completion reports
- Feature implementation summaries
- Bug fix documentation
- Session summaries

**Usage**: Understanding how features were implemented and decisions made.

---

## 🔍 Finding Documents

### By Date
Files are prefixed with dates when possible: `YYYY-MM-DD-description.md`

Sort by name to see chronological order.

### By Topic
Use directory structure:
- Deployment issues → `deployments/`
- Testing approaches → `testing/`
- Feature implementations → `implementations/`

### By Epic/Story
Look for filenames like: `epic-1.14-complete.md`

---

## 📋 Archive Guidelines

### What Goes Here
- ✅ Historical deployment logs
- ✅ Completed testing phase documentation
- ✅ Implementation session summaries
- ✅ Troubleshooting guides from specific issues
- ✅ Dated status reports

### What DOESN'T Go Here
- ❌ Current project status (use `../current-status.md`)
- ❌ Active development docs (use parent directory)
- ❌ User stories (use `../../stories/`)
- ❌ Code or SQL files (not documentation)

---

## 🗂️ Archive Maintenance

### When to Archive
Archive documents when:
- Deployment is complete and stable
- Testing phase is finished
- Implementation session is done
- Document is no longer actively referenced

### Cleanup Schedule
- **Monthly**: Review archived files for redundancy
- **Quarterly**: Compress very old archives (1+ year)
- **Annually**: Evaluate if ancient archives can be removed

---

## 📜 Historical Context

These archives provide valuable context about:
- **Decisions made**: Why certain approaches were chosen
- **Challenges faced**: Problems encountered during development
- **Solutions found**: How issues were resolved
- **Lessons learned**: What worked and what didn't

**Value**: Helps avoid repeating mistakes and informs future decisions.
