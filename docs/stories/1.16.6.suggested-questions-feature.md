# Story 1.16.6: Context-Aware Suggested Questions

**Epic**: Chat Experience & Navigation Improvements
**Status**: 📋 Planned
**Created**: 2025-10-22
**Priority**: HIGH
**Related**: Story 1.16.5 (Auto-Generate Chat Titles), Epic 1.14 (Chat Component Reorganization)

---

## Overview

Implement an intelligent suggested questions feature that dynamically generates relevant questions based on the user's chat history and available policy documents. The suggestions will be displayed below the chat input area, similar to modern AI chat applications (ChatGPT, Claude, Perplexity), helping users discover relevant policy information and improving engagement.

---

## Problem Statement

### Current State

**Issue**: Users must manually formulate questions without guidance

**Current Chat Interface**:
```
┌─────────────────────────────────────────┐
│ [Chat messages area]                    │
│                                         │
│ User: What is the leave policy?        │
│ AI: [Response about leave policy]      │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│ Type your message...          [Send]    │
│                                         │
└─────────────────────────────────────────┘
```

**Problems**:
- ❌ Users don't know what questions they can ask
- ❌ No discovery mechanism for related policies
- ❌ Users may miss relevant information
- ❌ Requires user to think of all follow-up questions
- ❌ Lower engagement and underutilization of system

### Desired State

**Improved Chat Interface with Suggested Questions**:
```
┌─────────────────────────────────────────┐
│ [Chat messages area]                    │
│                                         │
│ User: What is the leave policy?        │
│ AI: [Response about leave policy]      │

│ [What's the sick leave approval process?]│
│ [Can I carry over unused leave days?]  │
```

**Benefits**:
- ✅ Guides users to relevant information
- ✅ Increases engagement and discovery
- ✅ Reduces cognitive load (don't need to formulate every question)
- ✅ Exposes breadth of policy coverage
- ✅ Modern, familiar UX pattern
- ✅ Contextually relevant to conversation

## User Stories

### Story 1: First-Time User Sees Relevant Starting Questions

**As a** new user opening a chat session for the first time
**I want to** see suggested questions based on available policy documents
**So that** I know what topics I can ask about without having to guess

**Acceptance Criteria**:
- [ ] When chat session has no messages (empty state), display 3-5 suggested questions
- [ ] Questions are generated from available policy documents based on user's role
- [ ] Questions cover different policy areas (leave, benefits, safety, etc.)
- [ ] Clicking a question sends it as a user message
- [ ] Suggested questions disappear after first message is sent

**Example Suggested Questions (Empty Chat)**:
```
• What are the company's remote work policies?
• How do I request time off?
• What benefits am I eligible for?
• What is the performance review process?
```

---

### Story 2: User Sees Follow-Up Questions Based on Conversation

**As a** user having a conversation about policies
**I want to** see relevant follow-up questions after each AI response
**So that** I can explore related topics without thinking of every question myself

**Acceptance Criteria**:
- [ ] After each AI response, generate 3 contextually relevant follow-up questions
- [ ] Questions are based on current chat history AND available documents
- [ ] Questions should be natural follow-ups to the conversation
- [ ] Questions update dynamically after each exchange
- [ ] Questions are visually distinct from chat messages

**Example Flow**:
```
User: "What is the leave policy?"
AI: [Explains leave policy basics...]

Suggested questions:
• How many vacation days am I entitled to per year?
• What's the process for requesting sick leave?
• Can I use leave days for mental health?
```

---

### Story 3: Questions Disappear While User Is Typing

**As a** user typing my own question
**I want** suggested questions to get out of the way
**So that** I can focus on my custom question without visual clutter

**Acceptance Criteria**:
- [ ] Suggested questions are visible when input is empty
- [ ] Suggested questions fade out when user starts typing
- [ ] Questions reappear if user clears input (if still relevant)
- [ ] No layout shift when questions appear/disappear

## Technical Requirements

### 1. New Edge Function: `generate-suggested-questions`

**File**: `supabase/functions/generate-suggested-questions/index.ts`

**Purpose**: Generate contextually relevant questions based on chat history and available documents

**Input Parameters**:
```typescript
interface GenerateSuggestionsRequest {
  chatHistory: ChatMessage[];     // Current conversation
  sessionId: string;               // Chat session ID
  documentIds?: string[];          // Available document IDs (optional)
}

interface ChatMessage {
  role: 'user' | 'assistant';
  content: string;
  timestamp?: string;
}
```

**Output**:
```typescript
interface GenerateSuggestionsResponse {
  success: boolean;
  questions: string[];  // Array of 3-5 suggested questions
  error?: string;
}
```

**Implementation Requirements**:
- Use OpenAI API with gpt-4o-mini for cost efficiency
- Analyze last 3-4 message exchanges for context
- Query available documents to understand topic coverage
- Generate 3-5 natural, conversational questions
- Prioritize follow-up questions over tangential topics
- Ensure questions are answerable by available documents
- Response time target: < 2 seconds

**Prompt Strategy**:
```
System: You are a helpful assistant that generates relevant follow-up questions
for a policy Q&A system. Based on the conversation history and available policy
documents, suggest 3-5 questions the user might want to ask next. Questions
should be:
- Natural and conversational
- Relevant to the current topic
- Answerable by the available policy documents
- Progressively more detailed (start broad, get specific)

Available policy topics: [list from documents]
Conversation history: [chat history]

Generate 3-5 follow-up questions as a JSON array.
```

**Cost Optimization**:
- Only generate new questions after AI responses (not after every user message)
- Cache suggestions for 30 seconds to avoid duplicate calls
- Use streaming mode off (we need full response)
- Estimated cost: ~$0.0001 per generation

### 2. Frontend Hook: `useGenerateSuggestedQuestions`

**File**: `src/hooks/useGenerateSuggestedQuestions.ts`

**Purpose**: React Query hook to call edge function and manage question state

**Implementation**:
```typescript
import { useMutation } from '@tanstack/react-query';
import { supabase } from '@/integrations/supabase/client';

interface ChatMessage {
  role: 'user' | 'assistant';
  content: string;
  timestamp?: string;
}

interface GenerateSuggestionsParams {
  chatHistory: ChatMessage[];
  sessionId: string;
  documentIds?: string[];
}

export const useGenerateSuggestedQuestions = () => {
  const generateQuestions = useMutation({
    mutationFn: async ({
      chatHistory,
      sessionId,
      documentIds
    }: GenerateSuggestionsParams) => {
      const { data, error } = await supabase.functions.invoke(
        'generate-suggested-questions',
        {
          body: {
            chatHistory,
            sessionId,
            documentIds,
          },
        }
      );

      if (error) throw error;
      return data;
    },
  });

  return {
    generateQuestions,
    isGenerating: generateQuestions.isPending,
  };
};
```

### 4. Integration into ChatArea

**File**: `src/components/notebook/ChatArea.tsx`

**Changes Required**:

**State Management**:
```typescript
const [suggestedQuestions, setSuggestedQuestions] = useState<string[]>([]);
const [showSuggestions, setShowSuggestions] = useState(true);
const [isInputFocused, setIsInputFocused] = useState(false);

const { generateQuestions, isGenerating } = useGenerateSuggestedQuestions();
```

**Generate Questions After AI Response**:
```typescript
useEffect(() => {
  // Only generate after AI responses (when message count is even)
  const shouldGenerate =
    messages.length > 0 &&
    messages.length % 2 === 0 &&  // Even number = last message was AI
    !isGenerating;

  if (shouldGenerate) {
    const chatHistory = messages.slice(-6).map(msg => ({
      role: msg.message.type === 'human' ? 'user' as const : 'assistant' as const,
      content: typeof msg.message.content === 'string'
        ? msg.message.content
        : JSON.stringify(msg.message.content),
      timestamp: msg.message.created_at,
    }));

    generateQuestions.mutate(
      { chatHistory, sessionId: notebookId!, documentIds: sources.map(s => s.id) },
      {
        onSuccess: (data) => {
          setSuggestedQuestions(data.questions || []);
          setShowSuggestions(true);
        },
      }
    );
  }
}, [messages.length]);
```

**Hide/Show Logic**:
```typescript
// Hide when user is typing
const handleInputChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  setMessage(e.target.value);
  setShowSuggestions(e.target.value.length === 0);
};

// Send suggested question
const handleSuggestedQuestionClick = (question: string) => {
  setMessage(question);
  handleSendMessage(question);
  setShowSuggestions(false);
};
```

**JSX Placement** (below input, before footer):
```tsx
{/* Chat Input */}
<div className="p-6 border-t border-gray-200">
  <div className="max-w-4xl mx-auto">
    <Input
      value={message}
      onChange={handleInputChange}
      onFocus={() => setIsInputFocused(true)}
      onBlur={() => setIsInputFocused(false)}
    />

    {/* Suggested Questions */}
    {showSuggestions && suggestedQuestions.length > 0 && (
      <SuggestedQuestions
        questions={suggestedQuestions}
        onQuestionClick={handleSuggestedQuestionClick}
        isLoading={isGenerating}
        isVisible={message.length === 0}
      />
    )}
  </div>
</div>
```

## Risks & Mitigations

### Risk 1: Slow Generation Time
**Risk**: AI question generation takes >2 seconds, feels laggy
**Mitigation**:
- Use gpt-4o-mini (faster, cheaper)
- Show loading skeleton immediately
- Generate questions in background (don't block UI)
- Cache recent suggestions

### Risk 2: Irrelevant Questions
**Risk**: Generated questions not relevant to conversation
**Mitigation**:
- Carefully craft prompt with context
- Include document metadata in prompt
- A/B test different prompts
- Add user feedback mechanism ("Not helpful" button)

### Risk 3: Cost Concerns
**Risk**: Generating questions for every exchange is expensive
**Mitigation**:
- Only generate after AI responses (not user messages)
- Cache for 30 seconds
- Use smallest viable model (gpt-4o-mini)
- Estimated: $0.0001 per generation = $1 per 10,000 generations

### Risk 4: Layout Shift
**Risk**: Questions appearing/disappearing causes jarring layout shift
**Mitigation**:
- Reserve space with min-height
- Use smooth CSS transitions
- Fade in/out instead of instant show/hide
- Test on various screen sizes
