# Story 1.16.4: Outdated Policy Indicators

**Status**: ✅ Completed & Deployed
**Date**: October 21, 2025
**Priority**: High
**Related**: Policy Date Metadata Extraction (Story 2.3, 2.4)

## Overview

Implemented automated outdated policy detection and visual indicators throughout the application. Policies older than 18 months are automatically flagged with prominent badges in both the Dashboard and Sources sidebar, helping users quickly identify policies that may need review or updating.

## Feature Requirements

### Business Logic
- **Threshold**: Policies older than 18 months from current date are considered "outdated"
- **Source**: Uses `policyDate` field from `sources` table (format: "Month-Year", e.g., "February-2024")
- **Calculation**: Parses month-year format and calculates age in months
- **Display**: Shows visual indicator (badge) and age description

### Visual Design (Per UX Spec)
- **Color**: Amber/Warning color (`#FFB84D` - hh-warning from spec)
- **Icon**: AlertTriangle (Lucide React)
- **Badge Style**: Outlined badge with light amber background
- **Tooltip**: Shows exact age (e.g., "2 years, 3 months old")

## Implementation

### 1. Policy Date Utility Library

**File**: `src/lib/policyDateUtils.ts`

**Functions**:
- `parsePolicyDate()` - Parses "Month-Year" format to Date object
- `getPolicyAgeInMonths()` - Calculates age in months from today
- `isPolicyOutdated()` - Returns true if policy > 18 months old
- `getPolicyAgeDescription()` - Returns human-readable age ("2 years, 3 months old")
- `formatPolicyDate()` - Formats for display ("Feb 2024")

**Example Usage**:
```typescript
import { isPolicyOutdated, formatPolicyDate, getPolicyAgeDescription } from '@/lib/policyDateUtils';

const policyDate = "February-2024";

isPolicyOutdated(policyDate);           // true/false
formatPolicyDate(policyDate);           // "Feb 2024"
getPolicyAgeDescription(policyDate);    // "10 months old"
```

**Supported Formats**:
- "January-2024"
- "February-2024"
- "March-2025"
- Case-insensitive month names
- Handles both hyphen (-) and en-dash (–) separators

## User Experience Flow

### Dashboard View
1. User navigates to Dashboard
2. Document cards load with policy metadata
3. If `policyDate` is > 18 months old:
   - "Outdated" badge appears in amber
   - Badge includes warning icon
   - Hover shows full age description
4. Policy date displays below with age in parentheses

### Chat Sources Sidebar
1. User opens chat session
2. Sources sidebar loads linked documents
3. Each source shows:
   - Processing status badge (Ready/Processing/Failed)
   - Outdated badge (if applicable)
   - Policy date with calendar icon
   - Age description (if not outdated)
4. User can quickly identify old policies at a glance

## Database Schema

**Required Field**: `sources.policyDate`
- **Type**: TEXT (or VARCHAR)
- **Format**: "Month-Year" (e.g., "February-2024")
- **Nullable**: Yes (policies without dates won't show badges)
- **Example Values**:
  - "January-2023"
  - "March-2024"
  - "December-2025"

## Examples

### Policy Age Calculations

| Policy Date | Current Date | Age | Is Outdated? |
|-------------|--------------|-----|--------------|
| Feb 2024 | Oct 2025 | 20 months | ✅ Yes (>18) |
| May 2024 | Oct 2025 | 17 months | ❌ No (<18) |
| Jan 2023 | Oct 2025 | 33 months | ✅ Yes (>18) |
| Sep 2025 | Oct 2025 | 1 month | ❌ No |

### Age Descriptions

- 0 months → "Current month"
- 1 month → "1 month old"
- 11 months → "11 months old"
- 12 months → "1 year old"
- 13 months → "1 year, 1 month old"
- 18 months → "1 year, 6 months old"
- 24 months → "2 years old"
- 27 months → "2 years, 3 months old"

## Known Limitations

- **Date Precision**: Uses month-level granularity (not day-level)
- **Manual Entry**: Relies on policyDate being correctly entered during upload
- **Format Dependency**: Requires strict "Month-Year" format
- **No Validation**: Does not validate that policy content matches the date
- **Client-Side Only**: Calculation happens in UI, not database query
