# Story 1.7.2: Token Usage Tracking System

## Status: Draft

## Story

As a Company Operator,
I want to track token usage across all AI providers and models,
so that I can monitor costs and usage patterns for budget management.

## Context Source

- Source Document: Sprint Change Proposal - Epic 1.7
- Enhancement Type: Token usage tracking and monitoring
- Existing System Impact: New tracking system for AI token consumption

## Acceptance Criteria

1. Token usage is automatically tracked for all AI API calls (input/output tokens).
2. Usage data includes provider, model, timestamp, user, and token counts.
3. Real-time dashboard shows current usage, daily/weekly/monthly trends.
4. Cost calculation based on provider pricing (configurable rates).
5. Usage alerts when approaching limits (configurable thresholds).
6. Export functionality for usage reports (CSV/JSON format).
7. Historical data retention (configurable period, default 1 year).
8. Role-based access: Company Operators see company data, System Owners see all data.

## Dev Technical Guidance

### Token Usage Tracking Architecture

**Database Schema:**
- `token_usage` table for tracking all AI calls
- `usage_limits` table for company/user limits
- `provider_pricing` table for cost calculations
- Automatic tracking via middleware

**Tracking Implementation:**
- Intercept all AI API calls
- Extract token counts from responses
- Store usage data with metadata
- Calculate costs in real-time

**Dashboard Components:**
- Real-time usage display
- Historical trend charts
- Cost analysis views
- Export functionality

### UI Components Required

**Token Dashboard:**
- `TokenDashboard.tsx` - Main dashboard interface
- `UsageChart.tsx` - Historical usage charts
- `CostAnalysis.tsx` - Cost breakdown display
- `UsageAlerts.tsx` - Limit warning system
- `ExportReports.tsx` - Report generation

**Usage Tracking:**
- `TokenUsageTracker.tsx` - Real-time tracking
- `UsageLimits.tsx` - Limit configuration
- `ProviderPricing.tsx` - Pricing management
- `UsageHistory.tsx` - Historical data view

### API Endpoints Required

**Usage Tracking:**
- `POST /api/usage/track` - Record token usage
- `GET /api/usage/dashboard` - Dashboard data
- `GET /api/usage/history` - Historical usage
- `GET /api/usage/export` - Export reports

**Limits & Pricing:**
- `GET /api/usage/limits` - Get usage limits
- `PUT /api/usage/limits` - Update limits
- `GET /api/usage/pricing` - Get pricing data
- `PUT /api/usage/pricing` - Update pricing
