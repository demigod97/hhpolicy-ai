# Story 1.7.3: User Limits Configuration System

## Status: Draft

## Story

As a System Owner,
I want to configure usage limits for different user roles and individual users,
so that I can control costs and prevent abuse while ensuring fair resource allocation.

## Context Source

- Source Document: Sprint Change Proposal - Epic 1.7
- Enhancement Type: User limits and quota management
- Existing System Impact: New limits system for controlling AI usage

## Acceptance Criteria

1. System Owner can set default limits for each role (Board, Administrator, Executive, Company Operator).
2. Individual user limit overrides can be set by System Owner.
3. Limits include: daily token limit, monthly token limit, API calls per hour, cost limit per month.
4. Real-time enforcement of limits with graceful degradation (queue requests when approaching limits).
5. Limit violation notifications to users and System Owner.
6. Limit usage dashboard showing current usage vs limits for all users.
7. Bulk limit updates for multiple users at once.
8. Limit history tracking for audit purposes.

## Dev Technical Guidance

### User Limits Architecture

**Database Schema:**
- `user_limits` table for individual user limits
- `role_limits` table for default role limits
- `limit_violations` table for tracking violations
- `limit_history` table for audit trail

**Limit Enforcement:**
- Real-time limit checking before AI calls
- Graceful degradation when limits approached
- Queue system for rate limiting
- Notification system for violations

**Configuration UI:**
- Role-based limit configuration
- Individual user limit overrides
- Bulk limit updates
- Usage monitoring dashboard

### UI Components Required

**Limit Configuration:**
- `UserLimitsManager.tsx` - Main limits management
- `RoleLimitsConfig.tsx` - Role-based limits
- `IndividualLimitsConfig.tsx` - User-specific limits
- `BulkLimitsUpdate.tsx` - Bulk operations
- `LimitViolations.tsx` - Violation tracking

**Usage Monitoring:**
- `UsageLimitsDashboard.tsx` - Usage vs limits
- `LimitHistory.tsx` - Historical limit changes
- `ViolationReports.tsx` - Violation reports
- `LimitAlerts.tsx` - Limit warnings

### API Endpoints Required

**Limit Management:**
- `GET /api/limits/roles` - Get role limits
- `PUT /api/limits/roles` - Update role limits
- `GET /api/limits/users` - Get user limits
- `PUT /api/limits/users` - Update user limits
- `POST /api/limits/bulk` - Bulk limit updates

**Usage & Violations:**
- `GET /api/limits/usage` - Get usage vs limits
- `GET /api/limits/violations` - Get violations
- `POST /api/limits/check` - Check limit before call
- `GET /api/limits/history` - Get limit history
