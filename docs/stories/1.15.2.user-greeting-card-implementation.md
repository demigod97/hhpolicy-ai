### 2. Document Statistics Hook ✅
**File**: `src/hooks/useDocumentStats.tsx`

**Capabilities**:
- ✅ Fetches document statistics via React Query
- ✅ RLS-aware (respects role-based access)
- ✅ Provides four statistics:
  - **Total**: All accessible documents
  - **Recent**: Documents from last 7 days
  - **Uploaded**: Documents uploaded by current user
  - **Processing**: Documents currently being processed
- ✅ Auto-refresh with 30-second stale time
- ✅ Refetch on window focus
- ✅ Loading states handled

**Query Structure**:
```typescript
interface DocumentStats {
  total: number;
  recent: number;
  uploaded: number;
  processing: number;
}
```

## Technical Implementation

### Component Architecture

**UserGreetingCard.tsx**:
- Uses `useAuth()` for user information
- Uses `useUserRole()` for role-based display
- Uses `useDocumentStats()` for statistics
- Uses `useNavigate()` for navigation
- Dispatches custom events for upload dialog

**useDocumentStats Hook**:
- Powered by React Query
- Multiple parallel Supabase queries
- Optimistic caching strategy
- Error boundaries handled

### Styling

**shadcn Components Used**:
- `Card` and `CardContent` for container
- `Avatar` and `AvatarFallback` for user picture
- `Badge` for role display
- `Button` for quick actions
- `Separator` for visual division

**Tailwind Classes**:
- Role-specific color palettes
- Hover effects (`hover:shadow-md`)
- Transition animations (`transition-shadow duration-200`)
- Responsive grid (`grid-cols-3 gap-6`)
- Consistent spacing (`p-6`, `gap-4`)

### Role-Based Features

| Role | Quick Actions | Statistics Shown |
|------|--------------|------------------|
| **System Owner** | New Chat, Upload, Manage | Total, Recent, Uploads |
| **Company Operator** | New Chat, Upload, Manage | Total, Recent, Uploads |
| **Board** | New Chat | Total, Recent, Processing |
| **Executive** | New Chat | Total, Recent, Processing |
| **Administrator** | New Chat | Total, Recent, Processing |

## Notes

### Design Decisions

1. **Role Colors**: Used distinct, accessible colors for each role
2. **Avatar**: Initials fallback (no image upload needed)
3. **Statistics**: Three cards for balanced layout
4. **Quick Actions**: Right-aligned for visual hierarchy
5. **Event System**: Custom events for upload dialog (avoids prop drilling)

### Known Limitations

- No profile image upload (using initials)
- Statistics refresh every 30 seconds (not real-time)
- Upload button requires notebook ID
- Manage button requires admin role check in target page
