# Story 1.17.5: Dashboard Mobile Optimization - Collapsible Greeting & Scrollable Suggestions

## Status
Ready for Review
## Story

**As a** PolicyAi user (All Roles),
**I want** the dashboard to prioritize document visibility on mobile devices and enable smooth navigation through suggested questions,
**so that** I can efficiently access documents and explore AI-generated questions without excessive scrolling or navigation friction.

## Context

**Priority:** Critical
**Epic:** 1.17 - Dashboard & Chat Enhancements
**UX Audit Reference:** Completed by Sally (UX Expert) - 2025-10-29 - docs/ux-audits/dashboard-mobile-scrolling-audit.md
**Related Stories:** 1.17.4 (Mobile Citation Viewer)

### Problem Statement

This story addresses **two critical mobile UX issues** identified through user feedback and UX audit:

#### Problem 1: Dashboard Mobile Scrolling (Primary Issue)

On **mobile devices** (<1100px viewport), the UserGreetingCard component occupies **82% of the viewport** (767px on a 932px screen), leaving only 11% visible space for the DocumentTable. This forces users to scroll excessively to view their primary content (policy documents).

**Current Layout Breakdown:**
- Navigation: 7% (64px)
- UserGreetingCard: **82%** (767px) ⚠️
- DocumentTable: 11% (165px)
- Footer: ~4%

**Root Cause:**
- Stats grid uses `grid-cols-1` on mobile, causing 3 stat cards to stack vertically
- Large typography (`text-4xl`) and generous padding (`p-4`) optimized for desktop
- No mobile-specific condensed layout

**User Impact:**
- **High Frustration**: Excessive scrolling to reach documents (primary use case)
- **Poor First Impression**: Landing on dashboard shows greeting, not content
- **Accessibility Issues**: Screen readers announce 15+ elements before DocumentTable
- **Business Impact**: Reduced mobile engagement, poor mobile adoption

#### Problem 2: Suggested Questions Scroll Navigation (Secondary Issue)

In the chat interface (both desktop and mobile), the SuggestedQuestions component uses horizontal scroll (shadcn ScrollArea) but lacks:
- **Visual scroll indicators**: Users don't know more questions exist off-screen
- **Navigation controls**: No arrows/dots to guide scrolling
- **Touch affordance**: No visual cue that horizontal swiping is enabled
- **Keyboard navigation**: Cannot arrow-key through questions

**Current Implementation** (SuggestedQuestions.tsx:98-111):
```tsx
<ScrollArea className="w-full">
  <div className="flex gap-2 pb-2">
    {questions.map((question, index) => (
      <Button key={index} variant="outline" onClick={...}>
        {question}
      </Button>
    ))}
  </div>
</ScrollArea>
```

**User Impact:**
- **Discoverability**: Users miss questions beyond first 2-3 visible items
- **Accessibility**: Keyboard users cannot navigate suggestions
- **Mobile UX**: Swipe gesture not obvious; questions appear cut off

### Proposed Solutions

#### Solution 1: Collapsible UserGreetingCard (Primary)

Implement collapsible greeting card using shadcn Collapsible component:
- **Default State (Mobile)**: Collapsed - shows only greeting + role badge (~120px height)
- **Expanded State**: Full card with stats grid (current 767px design preserved)
- **Desktop Behavior**: Always expanded (zero regression)
- **User Control**: "View Stats" / "Hide Stats" toggle button
- **State Persistence**: localStorage saves user preference

**Expected Outcome:**
- Viewport allocation: 11% → **80%** for DocumentTable
- Scroll reduction: 767px → 120px to reach first document
- User empowerment: Optional stats viewing, content-first hierarchy

#### Solution 2: Enhanced Suggested Questions Navigation (Secondary)

Add visual navigation controls to SuggestedQuestions component:
- **Scroll Indicators**: Fade gradients on left/right edges when more questions exist
- **Arrow Navigation**: Left/Right chevron buttons for desktop
- **Swipe Hint**: Subtle animation or "Swipe for more" text on mobile
- **Keyboard Support**: Arrow keys to navigate between questions
- **Active Indicator**: Dots showing total questions and current scroll position

**Expected Outcome:**
- Improved discoverability of all suggestions
- Better accessibility for keyboard/screen reader users
- Clearer affordance for horizontal scroll interaction

## Acceptance Criteria

### Dashboard Mobile Optimization (Primary Feature)

1. **Collapsible Card - Mobile Default State**
   - UserGreetingCard defaults to collapsed on viewports <1100px
   - Collapsed state shows: Avatar, greeting, role badge, collapse toggle button
   - Collapsed height ≤150px (leaves 80%+ viewport for DocumentTable)

2. **Collapsible Card - Expanded State**
   - "View Stats" button expands card to show quick actions + stats grid
   - Expanded design identical to current layout (no visual regression)
   - "Hide Stats" button collapses card back to compact state

3. **Desktop Behavior Unchanged**
   - UserGreetingCard always fully expanded on viewports ≥1100px
   - No collapse trigger visible on desktop
   - Zero regression in desktop layout

4. **State Persistence**
   - User's collapse/expand preference saved in localStorage
   - Preference persists across page refreshes and sessions
   - Separate preferences for mobile/desktop (desktop always expanded)

5. **Smooth Animation**
   - Expand/collapse animation: 300ms CSS transition (shadcn default)
   - No layout jank or content shift
   - 60fps animation performance verified

6. **Touch Target Standards**
   - Collapse/expand toggle button ≥44x44px (iOS/Android standard)
   - Button uses `touch-action: manipulation` to prevent double-tap zoom

7. **Accessibility - Collapsible Card**
   - Toggle button has `aria-expanded`, `aria-controls` attributes
   - Screen reader announces "Stats section collapsed/expanded"
   - Keyboard navigation: Enter/Space to toggle
   - Focus trapped appropriately when expanded

### Suggested Questions Navigation Enhancement (Secondary Feature)

8. **Visual Scroll Indicators**
   - Fade gradients appear on left edge when scrolled right
   - Fade gradients appear on right edge when more questions exist off-screen
   - Gradients dynamically update based on scroll position

9. **Navigation Controls (Desktop)**
   - Left/Right chevron buttons appear when questions overflow container
   - Clicking arrows scrolls by one question width
   - Buttons disabled when at start/end of list

10. **Mobile Swipe Affordance**
    - First load shows subtle animation or "Swipe for more →" hint
    - Hint disappears after first swipe interaction
    - Questions beyond viewport show partial view (peek effect)

11. **Keyboard Navigation**
    - Arrow Left/Right keys navigate between question buttons
    - Tab key follows standard focus order
    - Enter/Space activates focused question button

12. **Active Question Indicator**
    - Dot indicators below questions show total count and current position
    - Dots update as user scrolls through questions

13. **Accessibility - Suggested Questions**
    - Screen reader announces "X of Y questions" when navigating
    - Focus visible indicator on all interactive elements
    - ARIA labels for navigation buttons ("Previous question", "Next question")

### Quality Requirements

14. **Zero Desktop Regression**
    - All existing desktop functionality unchanged
    - Dashboard layout on ≥1100px viewports identical to current
    - Chat interface desktop behavior unchanged

15. **Mobile Viewport Coverage**
    - Tested on iPhone SE (375x667px), iPhone 14 Pro (430x932px), Pixel 5 (393x851px)
    - First 3 document rows visible without scrolling on all devices
    - Suggested questions readable and navigable on all devices

16. **Performance**
    - Collapsible animation maintains 60fps
    - Scroll animations smooth (no jank)
    - No increased page load time

#### Pattern 2: shadcn Collapsible Component

```tsx
import { Collapsible, CollapsibleContent, CollapsibleTrigger } from '@/components/ui/collapsible';

<Collapsible open={isExpanded} onOpenChange={setIsExpanded}>
  {/* Always visible */}
  <div>Header content</div>

  <CollapsibleTrigger asChild>
    <Button>Toggle</Button>
  </CollapsibleTrigger>

  {/* Conditionally visible */}
  <CollapsibleContent>
    Hidden content
  </CollapsibleContent>
</Collapsible>
```

#### Pattern 3: localStorage State Persistence

```tsx
const [isExpanded, setIsExpanded] = useState(() => {
  // Desktop: always expanded
  if (isDesktop) return true;

  // Mobile: check localStorage
  const stored = localStorage.getItem('dashboard-greeting-expanded');
  return stored === 'true';
});

useEffect(() => {
  if (!isDesktop) {
    localStorage.setItem('dashboard-greeting-expanded', String(isExpanded));
  }
}, [isExpanded, isDesktop]);
```

### Scroll Detection Logic

```tsx
const [canScrollLeft, setCanScrollLeft] = useState(false);
const [canScrollRight, setCanScrollRight] = useState(false);
const scrollAreaRef = useRef<HTMLDivElement>(null);

const updateScrollState = () => {
  const el = scrollAreaRef.current;
  if (!el) return;

  setCanScrollLeft(el.scrollLeft > 0);
  setCanScrollRight(el.scrollLeft + el.clientWidth < el.scrollWidth);
};

useEffect(() => {
  updateScrollState();
  const el = scrollAreaRef.current;
  el?.addEventListener('scroll', updateScrollState);
  return () => el?.removeEventListener('scroll', updateScrollState);
}, [questions]);

const scrollLeft = () => {
  const el = scrollAreaRef.current;
  if (!el) return;
  const buttonWidth = questionRefs.current[0]?.offsetWidth || 200;
  el.scrollBy({ left: -buttonWidth, behavior: 'smooth' });
};

const scrollRight = () => {
  const el = scrollAreaRef.current;
  if (!el) return;
  const buttonWidth = questionRefs.current[0]?.offsetWidth || 200;
  el.scrollBy({ left: buttonWidth, behavior: 'smooth' });
};
```

### Keyboard Navigation Logic

```tsx
const handleKeyNavigation = (e: React.KeyboardEvent, currentIndex: number) => {
  if (e.key === 'ArrowLeft' && currentIndex > 0) {
    e.preventDefault();
    questionRefs.current[currentIndex - 1]?.focus();
    questionRefs.current[currentIndex - 1]?.scrollIntoView({
      behavior: 'smooth',
      block: 'nearest',
      inline: 'center'
    });
  }

  if (e.key === 'ArrowRight' && currentIndex < questions.length - 1) {
    e.preventDefault();
    questionRefs.current[currentIndex + 1]?.focus();
    questionRefs.current[currentIndex + 1]?.scrollIntoView({
      behavior: 'smooth',
      block: 'nearest',
      inline: 'center'
    });
  }
};
```
