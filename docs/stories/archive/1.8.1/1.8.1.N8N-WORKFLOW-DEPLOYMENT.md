**Date:** January 17, 2025
**Purpose:** Deploy role-based n8n chat workflows and configure Supabase secrets

## 🎯 Overview

PolicyAi uses role-based n8n workflows for RAG (Retrieval-Augmented Generation) chat functionality:

1. **Board Chat** - Access to ALL policy documents
2. **Executive Chat** - Access to Executive + Administrator documents (via `policyType` metadata filter)
3. **Admin Chat** - Access to Administrator documents only (via `policyType` metadata filter)

Currently, only Admin Chat is deployed. This guide shows how to deploy Board and Executive workflows.

## 🚀 Deployment Steps

### Step 1: Access n8n Instance

1. Navigate to: `https://n8n-prod.coralshades.ai/`
2. Login with your credentials

### Step 2: Import Board Chat Workflow

1. In n8n, click **"Workflows"** → **"Import from File"**
2. Select file: `n8n/PolicyAi - Board Chat.json`
3. Review the workflow:
   - **Webhook Trigger**: Note the webhook URL
   - **Supabase Vector Store Node**: Configured with NO metadata filter (accesses all documents)
   - **OpenAI Chat Model**: Uses `OPENAI_API_KEY` from credentials
   - **Response Format**: Returns `{ output: [{ text, citations }] }`

4. **Activate the workflow** (toggle in top-right)
5. **Copy the webhook URL** (shown in the Webhook trigger node)
   - Format: `https://n8n-prod.coralshades.ai/webhook/XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX`

### Step 3: Import Executive Chat Workflow

1. In n8n, click **"Workflows"** → **"Import from File"**
2. Select file: `n8n/PolicyAi - Executive Chat.json`
3. Review the workflow:

   - **Supabase Vector Store Node**: Configured with metadata filter:

4. **Activate the workflow** (toggle in top-right)
5. **Copy the webhook URL** (shown in the Webhook trigger node)

### Step 5: Update Supabase Secrets

Once workflows are deployed and tested, add the webhook URLs to Supabase secrets:

```bash
# Set Board Chat URL
supabase secrets set BOARD_CHAT_URL="https://n8n-prod.coralshades.ai/webhook/YOUR-BOARD-WEBHOOK-ID"

# Set Executive Chat URL
supabase secrets set EXECUTIVE_CHAT_URL="https://n8n-prod.coralshades.ai/webhook/YOUR-EXECUTIVE-WEBHOOK-ID"

# Verify secrets are set
supabase secrets list
```

## 🛡️ Security Notes

### Authorization Header
The Authorization header format is:
```
Authorization: Coral@123
```

**NOT:**
```
Authorization: Bearer Coral@123
Authorization: Basic Coral@123
```

This is a simple token-based authentication used by n8n webhooks.

### Webhook URL Security
- Webhook URLs contain UUIDs that act as secret tokens
- Do not expose webhook URLs in client-side code
- All requests go through Supabase Edge Functions (server-side)
- JWT authentication enforced before calling webhooks

## 🔧 Troubleshooting

### Issue: Workflow not receiving requests
**Check:**
1. Workflow is activated (toggle in top-right)
2. Webhook URL is correct in Supabase secrets
3. Authorization header is set: `Authorization: Coral@123`

**Debug:**
```bash
# Check Edge Function logs
supabase functions logs copilotkit-adapter --tail

# Check n8n execution logs in n8n UI
# Workflows → Your Workflow → Executions
```

### Issue: 401 Unauthorized from n8n
**Solution:** Check `NOTEBOOK_GENERATION_AUTH` secret value:
```bash
# Should be exactly: Coral@123
supabase secrets list | grep NOTEBOOK_GENERATION_AUTH
```

### Issue: Wrong documents returned
**Check:**
1. **Board Chat**: No metadata filter (should see ALL documents)
2. **Executive Chat**: Metadata filter `policyType: [administrator, executive]`
3. **Admin Chat**: Metadata filter `policyType: [administrator]`

**Verify in n8n:**
- Open workflow
- Check **Supabase Vector Store** node
- Look at "Filter" or "Metadata Filter" settings

### Issue: Fallback to NOTEBOOK_CHAT_URL
**This is expected behavior if:**
- `BOARD_CHAT_URL` or `EXECUTIVE_CHAT_URL` not set
- Edge function logs will show: `"BOARD_CHAT_URL not set, falling back to NOTEBOOK_CHAT_URL"`

**To fix:**
- Deploy Board/Executive workflows in n8n
- Set the secrets in Supabase
- Redeploy copilotkit-adapter
