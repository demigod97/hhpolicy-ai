## Executive Summary

This test design document validates the AG-UI Protocol implementation for PolicyAi, ensuring seamless integration with existing chat functionality while adding CopilotKit's advanced features. The design maintains backward compatibility with the current n8n webhook system and Supabase edge functions.

### Test Objectives

1. **Validate AG-UI Integration**: Ensure CopilotKit properly communicates with AI agents
2. **Maintain Current Functionality**: Verify existing chat features continue to work
3. **Test Role-Based Access**: Validate 5-tier role system (System Owner → General User)
4. **Verify Streaming**: Test real-time message streaming with citations
5. **Security Validation**: Ensure API key encryption and RLS policies work correctly

---

## Current System Analysis

### Existing Architecture

**Frontend Components:**
- `ChatArea.tsx` - Main chat interface with markdown rendering
- `useChatMessages.tsx` - Chat state management with Supabase realtime
- `MarkdownRenderer.tsx` - Message display with citation support

**Backend Services:**
- Supabase Edge Function: `send-chat-message`
- n8n Webhook: Role-based chat endpoints (admin/executive/board)
- Supabase Realtime: Message synchronization
- Database: `n8n_chat_histories` table with JSONB message storage

**Current Message Flow:**
```
User Input → ChatArea → useChatMessages → Edge Function → n8n Webhook → AI Response → Realtime → UI Update
```

### Current Features to Preserve

1. **Message Types:**
   - Human messages (plain text)
   - AI messages with citations
   - Structured citations with source references
   - Page number and line number tracking

2. **Citation System:**
   - Click-to-navigate citations
   - Source title display
   - PDF page highlighting
   - Citation excerpt display

3. **Real-time Updates:**
   - Supabase Realtime subscriptions
   - Automatic message synchronization
   - Optimistic UI updates

4. **Role-Based Chat:**
   - Different webhooks per role (admin/executive/board)
   - Role hierarchy enforcement
   - User role determination from JWT

## AG-UI Protocol Design

### Architecture Overview

**New Message Flow with AG-UI:**
```
User Input → ChatArea → CopilotKit Provider → AG-UI Protocol → AI Agent → Stream Response → UI Update
                ↓                                                                            ↑
           [Backward Compatible]                                                            |
                ↓                                                                            |
         useChatMessages → Edge Function → n8n Webhook → Realtime → [Parallel Update]
```

### CopilotKit Integration Points

#### 1. Provider Setup
```typescript
// In App.tsx or Layout Component
```

### Message Format Compatibility

#### Current Format (n8n):
```typescript
interface N8nMessageFormat {
  type: 'human' | 'ai';
  content: string | {
    segments: Array<{ text: string; citation_id?: number }>;
    citations: Array<{
      citation_id: number;
      source_id: string;
      source_title: string;
      source_type: string;
      page_number?: number;
      chunk_index?: number;
      excerpt?: string;
    }>;
  };
}
```

#### AG-UI Format (CopilotKit):
```typescript
interface AGUIMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  metadata?: {
    citations?: Citation[];
    segments?: MessageSegment[];
    sourceReferences?: string[];
  };
}
```

#### Transformation Layer:
```typescript
const transformN8nToAGUI = (n8nMsg: N8nMessageFormat): AGUIMessage => {
  // Convert n8n format to AG-UI format
  return {
    id: generateId(),
    role: n8nMsg.type === 'human' ? 'user' : 'assistant',
    content: extractContent(n8nMsg.content),
    metadata: extractMetadata(n8nMsg.content)
  };
};

const transformAGUIToN8n = (aguiMsg: AGUIMessage): N8nMessageFormat => {
  // Convert AG-UI format to n8n format for storage
  return {
    type: aguiMsg.role === 'user' ? 'human' : 'ai',
    content: formatContentWithMetadata(aguiMsg.content, aguiMsg.metadata)
  };
};
```

```typescript
  const instructions = {
    system_owner: "You are a strategic policy advisor. Focus on high-level insights, risk assessment, and board-level recommendations.",
    company_operator: "You are a company operations assistant. Focus on practical implementation and operational compliance.",
```

   - **Mitigation:** Implement robust reconnection logic
   - **Test:** Test Case 1.3, 6.1

2. **Message Format Transformation**
   - **Risk:** Data loss in format conversion
   - **Mitigation:** Comprehensive transformation tests
   - **Test:** Test Case 2.3, 4.2

3. **Role-Based Routing**
   - **Risk:** Messages routed to wrong webhook
   - **Mitigation:** Extensive role testing
   - **Test:** Test Case 3.1, 3.2

### Medium Risk Items

1. **Performance Under Load**
   - **Risk:** Slowdowns with many concurrent users
   - **Mitigation:** Load testing and optimization
   - **Test:** Test Case 7.1, 7.3

2. **Citation Preservation**
   - **Risk:** Citation metadata lost
   - **Mitigation:** Metadata validation tests
   - **Test:** Test Case 4.2, 4.3

## Rollback Plan

If critical issues discovered:

1. **Immediate:** Disable AG-UI integration via feature flag
2. **Fallback:** Revert to pure n8n webhook system
3. **Investigation:** Debug issues in staging environment
4. **Fix:** Apply fixes and re-test
5. **Re-deploy:** Gradual rollout with monitoring

**Rollback Trigger Conditions:**
- > 10% error rate in production
- Data loss or corruption detected
- Security vulnerability discovered
- Performance degradation > 50%
