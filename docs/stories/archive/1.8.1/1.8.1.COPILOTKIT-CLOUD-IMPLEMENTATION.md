#### 2. Backend Adapter - copilotkit-adapter Edge Function
**File:** `supabase/functions/copilotkit-adapter/index.ts` (new)

**Purpose:** Bridge CopilotKit Cloud with existing n8n workflows

**Features:**
- ✅ JWT authentication via Supabase
- ✅ Role hierarchy detection (system_owner > board > company_operator > executive > administrator)
- ✅ Role-based instructions injection
- ✅ n8n webhook routing by role
- ✅ Message format transformation (CopilotKit ↔ n8n)
- ✅ Error handling with fallback

**Role-Based Routing:**
```typescript
switch (effectiveRole) {
  case 'system_owner':
  case 'board':
    webhookUrl = BOARD_CHAT_URL;
    break;
  case 'company_operator':
  case 'executive':
    webhookUrl = EXECUTIVE_CHAT_URL;
    break;
  case 'administrator':
  default:
    webhookUrl = NOTEBOOK_CHAT_URL;
    break;
}
```

**Role Instructions:** Inline role-based AI behavior definitions for each of the 5 roles

## 🔒 Security Considerations

### ✅ Implemented Security

1. **Authentication**
   - JWT validation via Supabase in copilotkit-adapter
   - CopilotKit Cloud publicApiKey for API access
   - User identity verified before role detection

2. **Authorization**
   - Role-based access control (RBAC) via user_roles table
   - Role hierarchy enforced in adapter
   - n8n workflows filter by policyType metadata

3. **Data Protection**
   - HTTPS/TLS for all requests
   - Supabase RLS policies enforce document access
   - No sensitive data in frontend logs

4. **API Key Management**
   - CopilotKit publicApiKey in environment variables
   - n8n webhook auth tokens in Supabase secrets
   - OpenAI API key never exposed to frontend
