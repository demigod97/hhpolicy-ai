# CopilotKit Open Source Deployment Guide

## Story 1.8.1: AG-UI Protocol Implementation

**Date:** January 17, 2025
**Status:** Implementation Complete - Ready for Deployment
**Mode:** CopilotKit Open Source (Self-Hosted)

## Overview

This guide covers deploying the CopilotKit open source runtime as a Supabase Edge Function, enabling AG-UI protocol support without Copilot Cloud dependency.

### Architecture

```
Frontend (React + Vite)
    ↓
CopilotKit Provider (runtimeUrl)
    ↓
Supabase Edge Function: copilotkit-runtime
    ↓
OpenAI/Anthropic API
    ↓
Response → Frontend
```

---

## Prerequisites

✅ **Supabase Project** with Edge Functions enabled
✅ **OpenAI API Key** (or Anthropic for Claude models)
✅ **Supabase CLI** installed and configured

## Step 1: Configure Environment Variables

### Local Development (.env.local)

```env
# Supabase Configuration
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key_here

# CopilotKit Open Source
VITE_COPILOTKIT_RUNTIME_URL=http://localhost:54321/functions/v1/copilotkit-runtime
```

### Supabase Edge Function Secrets

```bash
# Set OpenAI API key
supabase secrets set OPENAI_API_KEY=sk-your-openai-key-here

# Set LLM configuration
supabase secrets set COPILOTKIT_LLM_PROVIDER=openai
supabase secrets set COPILOTKIT_LLM_MODEL=gpt-4-turbo-preview

# Optional: Anthropic API key for Claude models
supabase secrets set ANTHROPIC_API_KEY=sk-ant-your-key-here
```


## Step 5: Frontend Integration

### App.tsx Configuration

The CopilotKit Provider is already configured in `src/App.tsx`:

```typescript
<CopilotKit
  runtimeUrl={import.meta.env.VITE_COPILOTKIT_RUNTIME_URL}
  showDevConsole={import.meta.env.DEV}
>
  {/* App content */}
</CopilotKit>
```

### Using in Components

```typescript

const MyComponent = () => {

    messages,
    sendMessage,
    isLoading

    notebookId: 'notebook-id',
    userRole: 'administrator',
```

## Monitoring & Debugging

### View Edge Function Logs

```bash
# Real-time logs
supabase functions logs copilotkit-runtime --tail

# Recent logs
supabase functions logs copilotkit-runtime
```

### Common Issues

#### 1. **401 Unauthorized**

**Cause:** Invalid or missing authorization token

**Solution:**
- Ensure `Authorization: Bearer <token>` header is present
- Verify token is valid Supabase auth token
- Check RLS policies on `user_roles` table

#### 2. **500 Internal Server Error**

**Cause:** Missing OpenAI API key or invalid configuration

**Solution:**
```bash
# Verify secrets are set
supabase secrets list

# Re-deploy function
supabase functions deploy copilotkit-runtime --no-verify-jwt
```

#### 3. **CORS Errors**

**Cause:** Frontend origin not allowed

**Solution:**
- Verify CORS headers in Edge Function
- Check `Access-Control-Allow-Origin` is set to `*` or specific origin

## Performance Optimization

### Edge Function Cold Starts

- **Warm-up:** First request may be slow (~2-5s)
- **Subsequent requests:** Fast (<1s)
- **Solution:** Implement keep-alive pings from frontend

### Rate Limiting

Add rate limiting to Edge Function:

```typescript
// In copilotkit-runtime/index.ts
const rateLimiter = new RateLimiter({
  maxRequests: 100,
  windowMs: 60000 // 100 requests per minute
});
```

## Rollback Plan

If AG-UI causes issues, disable by:

### 1. Feature Flag

```typescript
// In .env
VITE_ENABLE_AG_UI=false
```

### 2. Fallback to n8n

The `useAGUIChatMessages` hook has built-in fallback:

```typescript
{
  enableDualMode: true,  // Sends to both systems
  enableFallback: true   // Falls back to n8n on AG-UI error
}
```
