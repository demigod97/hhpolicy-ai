## Key Design Decisions

### 1. Backward Compatibility ✅
**Decision:** Run AG-UI alongside existing n8n system during migration

**Rationale:**
- Zero downtime for users
- Gradual validation of new system
- Easy rollback if issues arise
- Compare performance metrics

**Implementation:**
```typescript
// Dual-mode sending
const sendMessage = async (content: string) => {
  // Primary: AG-UI (new)
  await copilotChat.appendMessage({ role: 'user', content });
  
  // Backup: n8n webhook (existing)
  await supabase.functions.invoke('send-chat-message', { ... });
};
```

---

### 2. Message Format Transformation ✅
**Decision:** Create bidirectional transformers between n8n and AG-UI formats

**Rationale:**
- Preserve existing database structure
- Maintain citation system compatibility
- Enable seamless transition
- Support both systems simultaneously

**Components:**
- `transformN8nToAGUI()` - Convert stored messages to AG-UI format
- `transformAGUIToN8n()` - Convert AG-UI messages for storage
- `MessageTransformer` service - Handle all conversions

---

### 3. Role-Based Instructions ✅
**Decision:** Map 5-tier role system to AG-UI instructions

**Mapping:**
```typescript
{
  system_owner: "Strategic policy advisor, board-level insights",
  company_operator: "Operations assistant, practical compliance",
  executive: "Executive assistant, concise summaries",
  administrator: "Full system access, comprehensive assistant",
  policy_analyst: "Research-focused, detailed analysis",
  general_user: "Basic assistance, accessible policies"
}
```

---

### 4. Citation Preservation ✅
**Decision:** Maintain full citation metadata through AG-UI transformation

**Preserved Fields:**
- `citation_id` - Citation number
- `source_id` - Source document ID
- `source_title` - Document name
- `source_type` - pdf/text/website/youtube/audio
- `page_number` - PDF page reference
- `chunk_index` - Chunk location
- `chunk_lines_from` - Start line
- `chunk_lines_to` - End line
- `excerpt` - Citation text

---

### 5. Feature Flag System ✅
**Decision:** Use localStorage + gradual rollout for migration

**Rollout Strategy:**
- Week 1: 10% → 25% → 50%
- Week 2: 75% → 100%
- Monitor error rates at each stage
- Rollback capability at any point

## Integration Points

### 1. Current System (Preserved)
```
User → ChatArea → useChatMessages → Supabase Edge Function → n8n Webhook → AI
                                                                              ↓
                     ← Realtime ← n8n_chat_histories ← Response stored ←  ←  ←
```

### 2. New AG-UI System (Added)
```
User → ChatArea → CopilotKit → AG-UI Protocol → AI Agent → Streaming Response
                      ↓                                           ↓
                   Store in DB → n8n_chat_histories → Realtime → UI Update
```

### 3. Dual Mode (Migration)
```
User Input
    ↓
    ├─→ AG-UI (Primary)
    │      ↓
    │   AI Response (streamed)
    │      ↓
    │   Display in UI
    │
    └─→ n8n (Backup)
           ↓
        Webhook
           ↓
        Store in DB
```
