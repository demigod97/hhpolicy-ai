## Overview

Successfully transitioned from modal-based citation preview to an integrated sidebar citation viewer that displays full document content with highlighted cited sections. The implementation uses shadcn components for a polished UI/UX and maintains the existing sources list functionality.

## What Was Implemented

### 1. **DocumentViewer Component** (`src/components/chat/DocumentViewer.tsx`)

A new component that replaces the previous CitationPreview modal with a sidebar-friendly design:

**Key Features**:
- **Full Document Display**: Fetches ALL chunks for a source document from the `documents` table
- **Markdown Rendering**: Renders document content with proper formatting (headers, bold text, paragraphs)
- **Citation Highlighting**: Highlights the specific cited section with yellow background, border, and badge
- **Auto-Scroll**: Automatically scrolls to the highlighted citation when opened
- **Tabs Interface**:
  - **Full Document Tab**: Shows complete document text with citation highlighted
  - **PDF View Tab**: Embeds PDFViewer component for viewing original PDF
- **Close Button**: Allows users to dismiss the citation view

**Technical Implementation**:
```typescript
// Fetches all chunks and concatenates into full document
const chunks = docs.map((doc: any) => ({
  content: doc.content || '',
  linesFrom: doc.metadata?.loc?.lines?.from || 0,
  linesTo: doc.metadata?.loc?.lines?.to || 0,
}));

// Highlights cited chunk
const isCited = chunk.linesFrom === linesFrom && chunk.linesTo === linesTo;
```

### Auto-Scroll to Citation (Fixed - Oct 21)
**Original Implementation** (caused UI locking):
```typescript
// BAD - scrolls entire page
highlightRef.current.scrollIntoView({
  behavior: 'smooth',
  block: 'center',
});
```

**Fixed Implementation** (contained scroll):
```typescript
const scrollAreaRef = useRef<HTMLDivElement>(null);

useEffect(() => {
  if (highlightRef.current && documentData && scrollAreaRef.current) {
    // Wait for layout to settle
    const timer = setTimeout(() => {
      // Get the specific ScrollArea viewport
      const scrollContainer = scrollAreaRef.current?.querySelector('[data-radix-scroll-area-viewport]');

      if (scrollContainer && highlightRef.current) {
        // Calculate position within scroll container
        const containerTop = scrollContainer.getBoundingClientRect().top;
        const elementTop = highlightRef.current.getBoundingClientRect().top;
        const offset = elementTop - containerTop - 100; // 100px from top

        // Scroll ONLY the container, not the page
        scrollContainer.scrollTo({
          top: scrollContainer.scrollTop + offset,
          behavior: 'smooth',
        });
      }
    }, 300); // Delay prevents race conditions

    return () => clearTimeout(timer);
  }
}, [documentData]);
```

**Why the fix was needed:**
- Original `scrollIntoView()` scrolled parent containers causing tabs/headers to disappear
- UI became locked/unresponsive
- Users experienced white screens requiring page refresh
- Added `overflow-hidden` constraints throughout component tree
- Targeted specific ScrollArea viewport instead of entire page
