# Story 1.19.3: Enhanced Document Upload Flow

**Epic**: 1.19 - Template Library & Enhanced Document Upload
**Status**: 📋 Planned
**Created**: 2025-11-28
**Priority**: HIGH
**Story Points**: 13
**Estimated Effort**: 4-5 days
**Dependencies**: Story 1.19.1 (Template Infrastructure)

---

## Overview

Transform the current single-step document upload into a comprehensive multi-step wizard that collects rich metadata matching the policy template structure. This enables better searchability, compliance tracking, and role-based access control while providing a guided experience for Company Operators uploading policy documents.

---

## Business Requirements

**As a** Company Operator
**I want to** upload policy documents with comprehensive metadata
**So that** documents are properly categorized, searchable, and accessible to the right roles

**Value**:
- Improved RAG search accuracy through structured metadata
- Better compliance tracking with policy dates and ownership
- Enhanced user experience with guided upload process
- Consistent data quality across all policy documents

### 4. Integration with Document Processing

**Updated Upload Flow**:
```typescript
async function handleEnhancedUpload(metadata: DocumentMetadata, file: File) {
  // 1. Upload file to Supabase Storage
  const filePath = await uploadFileToStorage(file);

  // 2. Create source record with enhanced metadata
  const { data: source, error } = await supabase
    .from('sources')
    .insert({
      notebook_id: notebookId,
      title: metadata.title,
      file_path: filePath,
      file_name: file.name,
      file_size: file.size,

      // Enhanced metadata
      document_category: metadata.document_category,
      access_level: metadata.access_level,
      policy_summary: metadata.policy_summary,
      attributed_to: metadata.attributed_to,
      eligibility: metadata.eligibility,
      policy_information: metadata.policy_information,
      policy_content: metadata.policy_content,
      policy_notes: metadata.policy_notes,
      policy_comments: metadata.policy_comments,
      executive_notes: metadata.executive_notes,
      policy_effective_date: metadata.policy_effective_date,
      policy_last_updated: metadata.policy_last_updated,

      processing_status: 'pending',
      uploaded_by_user_id: userId,
    })
    .select()
    .single();

  // 3. Trigger N8N processing with metadata context
  await triggerDocumentProcessing(source.id, metadata);

  return source;
}
```

### 5. Form Validation

**Required Fields**:
- File (must be .doc, .docx, or .pdf)
- Title (1-200 chars)
- Document category
- Access level
- Policy summary (1-500 chars)
- Attributed to

**Optional Fields**:
- All other fields can be empty

**Validation Rules**:
```typescript
const validationSchema = {
  file: yup.mixed().required('File is required'),
  title: yup.string().min(1).max(200).required('Title is required'),
  document_category: yup.string().oneOf(['policy', 'process', 'checklist']).required(),
  access_level: yup.string().oneOf(['general', 'executive', 'board']).required(),
  policy_summary: yup.string().min(1).max(500).required('Summary is required'),
  attributed_to: yup.string().required('Attribution is required'),
  policy_effective_date: yup.date().nullable(),
  policy_last_updated: yup.date().nullable(),
};
```
