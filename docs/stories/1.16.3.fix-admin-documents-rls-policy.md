# Story 1.16.3: Fix Administrator Documents RLS Policy

**Status**: ✅ Migration Ready (Needs Apply)
**Date**: October 21, 2025
**Priority**: High
**Related**: Story 1.16.2 (Sidebar Citation Viewer)

## Bug Report

**Issue**: Admin role cannot view cited sources and documents in chat window

**Symptoms**:
- Administrators click on citations in chat messages
- DocumentViewer loads in sidebar but shows "No document content found"
- Sources list in sidebar is empty or incomplete
- Only documents from sources the admin personally uploaded are visible
- Citations referencing documents uploaded by other administrators fail to load

## Root Cause Analysis

The `documents` table has an overly restrictive RLS policy for administrators:

```sql
-- Current policy: administrators_manage_documents
USING (
  is_administrator()
  AND EXISTS (
    SELECT 1 FROM sources s
    WHERE s.id = documents.source_id
    AND s.uploaded_by_user_id = auth.uid()  -- ❌ PROBLEM: Only own uploads
  )
)
```

This policy requires administrators to be the uploader of the source, which breaks:
- **Citations**: Admin A cannot see citations from documents uploaded by Admin B
- **DocumentViewer**: Cannot load full document text for sources uploaded by others
- **SourcesSidebar**: Sources list incomplete for collaborative admin environments

## The Fix

Create a separate SELECT-only policy that allows administrators to view ALL administrator-targeted documents:

```sql
CREATE POLICY "administrators_view_admin_documents"
ON documents
FOR SELECT
TO authenticated
USING (
  is_administrator()
  AND EXISTS (
    SELECT 1
    FROM sources s
    WHERE s.id = documents.source_id
    AND s.target_role = 'administrator'  -- ✅ Any admin-targeted source
  )
);
```

## Policy Separation Strategy

After the fix, administrators will have **two policies**:

### 1. **administrators_view_admin_documents** (NEW - SELECT)
- **Purpose**: Allow admins to READ all administrator-targeted documents
- **Operations**: SELECT only
- **Scope**: Any source with `target_role = 'administrator'`
- **Use Case**: Citations, DocumentViewer, SourcesSidebar

### 2. **administrators_manage_documents** (EXISTING - INSERT/UPDATE/DELETE)
- **Purpose**: Allow admins to MODIFY only their own uploaded documents
- **Operations**: INSERT, UPDATE, DELETE
- **Scope**: Sources uploaded by the current admin
- **Use Case**: Document processing, updates, deletions

This separation ensures:
- ✅ Admins can view all shared admin documents (fixes citation bug)
- ✅ Admins can only modify documents they uploaded (maintains data integrity)
- ✅ Follows principle of least privilege for write operations

## Impact Assessment

**Before Fix**:
- Admins could only collaborate on documents they personally uploaded
- Citations broke in multi-admin environments
- DocumentViewer non-functional for shared documents
- Poor UX for admin teams

**After Fix**:
- All admins can view all administrator-targeted documents
- Citations work across admin uploads
- DocumentViewer functional for all admin documents
- Admins can still only modify their own uploads (security maintained)

## Security Considerations

**Data Access**:
- ✅ Admins can read all `target_role = 'administrator'` documents
- ✅ Admins cannot read executive-only or other role documents
- ✅ Admins can only modify documents they uploaded
- ✅ Role-based access control maintained via `target_role` filtering

**No Breaking Changes**:
- Existing policies remain unchanged
- Only adds additional SELECT permission
- No impact on INSERT/UPDATE/DELETE operations
- Backwards compatible with existing admin workflows
