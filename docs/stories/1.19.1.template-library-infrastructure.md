## Overview

Establish the backend infrastructure for a policy template management system that enables Company Operators to upload, manage, and distribute standardized Word document templates (Policy, Process, Checklist) across three access levels (General, Executive, Board). This foundational story creates the database schema, storage system, and APIs necessary for the template library feature.

---

## Business Requirements

### Why This Feature Matters

**Current State**:
- ❌ No standardized templates for policy document creation
- ❌ Users create policies from scratch without guidance
- ❌ Inconsistent document structure affects searchability
- ❌ No way to ensure proper metadata is collected
- ❌ Company Operators manually distribute templates via email/SharePoint

**Desired State**:
- ✅ Centralized template library with 9 standardized templates
- ✅ Templates accessible directly within PolicyAi application
- ✅ Company Operators can upload and manage templates
- ✅ Templates categorized by Type (Policy/Process/Checklist) and Access Level (General/Executive/Board)
- ✅ Templates stored securely in Supabase Storage with proper access control

**Business Value**:
- 📈 Improved policy document consistency and quality
- 📈 Reduced time to create new policies (download template vs. start from scratch)
- 📈 Better RAG search results due to standardized document structure
- 📈 Enhanced compliance through template-enforced sections
- 📈 Simplified onboarding for new policy creators

## Technical Requirements

### 1. Database Schema

#### Create `templates` Table

```sql
-- Migration: 20251128000001_create_templates_table.sql

-- Enable UUID extension if not already enabled
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Create templates table
CREATE TABLE IF NOT EXISTS public.templates (
    id uuid PRIMARY KEY DEFAULT uuid_generate_v4(),

    -- Template identification
    title text NOT NULL,
    description text,

    -- Categorization
    template_type text NOT NULL CHECK (template_type IN ('policy', 'process', 'checklist')),
    access_level text NOT NULL CHECK (access_level IN ('general', 'executive', 'board')),

    -- File storage
    file_path text NOT NULL,                    -- Path in Supabase Storage
    file_name text NOT NULL,                    -- Original filename
    file_size integer,                          -- Size in bytes
    storage_bucket text DEFAULT 'templates' NOT NULL,

    -- Metadata
    version integer DEFAULT 1,                  -- Template version
    is_active boolean DEFAULT true,             -- Soft delete flag

    -- Audit fields
    uploaded_by uuid REFERENCES auth.users(id) ON DELETE SET NULL,
    created_at timestamptz DEFAULT now() NOT NULL,
    updated_at timestamptz DEFAULT now() NOT NULL,

    -- Constraints
    CONSTRAINT unique_template_type_level UNIQUE(template_type, access_level, is_active)
    -- Ensures only one active template per type/level combination
);

-- Create index for common queries
CREATE INDEX idx_templates_type_level ON public.templates(template_type, access_level) WHERE is_active = true;
CREATE INDEX idx_templates_uploaded_by ON public.templates(uploaded_by);

-- Add updated_at trigger
CREATE OR REPLACE FUNCTION public.update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = now();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER update_templates_updated_at
    BEFORE UPDATE ON public.templates
    FOR EACH ROW
    EXECUTE FUNCTION public.update_updated_at_column();

-- Add comments for documentation
COMMENT ON TABLE public.templates IS 'Stores metadata for downloadable policy document templates';
COMMENT ON COLUMN public.templates.template_type IS 'Type of template: policy, process, or checklist';
COMMENT ON COLUMN public.templates.access_level IS 'Access level: general (administrator), executive, or board';
COMMENT ON COLUMN public.templates.is_active IS 'Soft delete flag - only active templates shown to users';
```

### 2. Row Level Security (RLS) Policies

```sql
-- Enable RLS on templates table
ALTER TABLE public.templates ENABLE ROW LEVEL SECURITY;

-- Policy 1: All authenticated users can view active templates
CREATE POLICY "All users can view active templates"
    ON public.templates
    FOR SELECT
    TO authenticated
    USING (is_active = true);

-- Policy 2: Company Operators and System Owners can insert templates
CREATE POLICY "Company Operators can insert templates"
    ON public.templates
    FOR INSERT
    TO authenticated
    WITH CHECK (
        EXISTS (
            SELECT 1 FROM public.user_roles
            WHERE user_id = auth.uid()
            AND role IN ('company_operator', 'system_owner')
        )
    );

-- Policy 3: Company Operators and System Owners can update templates
CREATE POLICY "Company Operators can update templates"
    ON public.templates
    FOR UPDATE
    TO authenticated
    USING (
        EXISTS (
            SELECT 1 FROM public.user_roles
            WHERE user_id = auth.uid()
            AND role IN ('company_operator', 'system_owner')
        )
    )
    WITH CHECK (
        EXISTS (
            SELECT 1 FROM public.user_roles
            WHERE user_id = auth.uid()
            AND role IN ('company_operator', 'system_owner')
        )
    );

-- Policy 4: System Owners can delete (soft delete by setting is_active = false)
CREATE POLICY "System Owners can delete templates"
    ON public.templates
    FOR UPDATE
    TO authenticated
    USING (
        EXISTS (
            SELECT 1 FROM public.user_roles
            WHERE user_id = auth.uid()
            AND role = 'system_owner'
        )
    );
```

### 3. Storage RLS Policies (SQL)

```sql
-- Storage policy: All authenticated users can read templates
CREATE POLICY "Authenticated users can read templates"
ON storage.objects FOR SELECT
TO authenticated
USING (bucket_id = 'templates');

-- Storage policy: Company Operators can upload templates
CREATE POLICY "Company Operators can upload templates"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (
    bucket_id = 'templates'
    AND EXISTS (
        SELECT 1 FROM public.user_roles
        WHERE user_id = auth.uid()
        AND role IN ('company_operator', 'system_owner')
    )
);

-- Storage policy: Company Operators can update templates
CREATE POLICY "Company Operators can update templates"
ON storage.objects FOR UPDATE
TO authenticated
USING (
    bucket_id = 'templates'
    AND EXISTS (
        SELECT 1 FROM public.user_roles

);
```
