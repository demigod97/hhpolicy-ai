```typescript
const { data: sources, error } = await supabase

  .eq('notebook_id', notebookId)
  .eq('processing_status', 'completed')
  .or(`title.ilike.%${query}%,content.ilike.%${query}%`)
```

## ⚠️ Known Issues & Limitations

### Critical Issue: Chat UI Failure

**Problem**: When AG-UI mode is enabled, sending messages causes chat UI to fail to load

**Symptoms**:
- Chat input becomes unresponsive
- No error messages displayed to user
- Console may show JSON parsing errors
- Legacy chat works perfectly

**Impact**: ⚠️ **HIGH** - AG-UI mode is not usable in production

**Workaround**:
```javascript
// Disable AG-UI mode
localStorage.removeItem('enableAGUI');
// Reload page
window.location.reload();
```

**Root Cause**: Unknown - requires further investigation
- Possible SSE parsing issues
- Potential message format mismatches
- May be related to CopilotKit version compatibility

### Other Limitations

1. **Action Testing**: Actions visible in Inspector but not runtime-tested
2. **Vector Search**: Not implemented (using basic text search)
3. **OpenAI Integration**: Uses N8N workflows instead of direct OpenAI calls
4. **Performance**: Not optimized for large document sets

## 🔍 Technical Implementation Details

### Architecture

```
┌─────────────────────────────────────────────────────────┐
│                     Frontend (React)                    │
│                                                           │
│  ┌────────────────┐          ┌─────────────────────┐   │
│  │  Legacy Chat   │          │   AG-UI Chat        │   │
│  │  (Working ✅)  │          │   (Broken ⚠️)       │   │
│  └────────┬───────┘          └─────────┬───────────┘   │
│           │                             │               │
│           │                             │               │
│  ┌────────▼───────────────────────────▼───────────┐   │
│  │      Feature Flag: enableAGUI                   │   │
│  │      - false = Legacy (N8N)                     │   │
│  │      - true = AG-UI (CopilotKit)               │   │
│  └────────┬───────────────────────┬────────────────┘   │
│           │                       │                     │
└───────────┼───────────────────────┼─────────────────────┘
            │                       │
            │                       │
  ┌─────────▼────────┐    ┌────────▼──────────────────┐
  │  N8N Webhooks    │    │ Supabase Edge Function    │
  │  - Admin Chat    │    │ copilotkit-runtime        │
  │  - Exec Chat     │    │ - GraphQL                 │
  │  - Board Chat    │    │ - SSE Streaming           │
  └─────────┬────────┘    │ - OpenAI Integration      │
            │             └───────────────────────────┘
            │
  ┌─────────▼────────────────────────────────────────┐
  │           Supabase Database                      │
  │  - sources (policy documents)                    │
  │  - documents (vector embeddings)                 │
  │  - n8n_chat_histories (messages)                │
  │  - user_roles (RBAC)                            │
  └──────────────────────────────────────────────────┘
```

### Data Flow

**Legacy Chat (Working)**:
1. User types message → ChatArea component
2. Message sent to Supabase function `send-chat-message`
3. Function routes to role-based N8N webhook
4. N8N processes with OpenAI + RAG
5. Response streamed back to frontend
6. Message saved to `n8n_chat_histories`

**AG-UI Chat (Not Working)**:
1. User types message → ChatAreaCopilotKit component
2. Message sent via `useCopilotChat` hook
3. Request to `copilotkit-runtime` Edge Function
4. ⚠️ **FAILURE POINT** - Chat UI doesn't load properly
5. (Expected: SSE streaming response)
6. (Expected: Display in chat interface)

## 💡 Lessons Learned

### What Worked Well

1. **Dual-Mode Architecture**: Feature flag approach allowed safe experimentation
2. **Incremental Implementation**: Building components separately helped isolation
3. **Documentation First**: Detailed planning made implementation smoother
4. **Supabase Integration**: Edge Functions deployed successfully
5. **Action Handler Pattern**: Clean separation of concerns

### What Didn't Work

1. **CopilotKit Compatibility**: Version/configuration issues unclear
2. **SSE Handling**: Custom vs built-in implementation confusion
3. **Testing Earlier**: Should have tested AG-UI chat sooner
4. **Complexity**: Too many moving parts (CopilotKit + N8N + Supabase)

### Key Insights

1. **Legacy System Value**: Don't deprecate working systems until replacement proven
2. **Feature Flags Essential**: Allow safe rollback when issues arise
3. **Integration Testing**: Need proper test environment for AG-UI
4. **Vendor Documentation**: CopilotKit docs incomplete for self-hosted scenarios
