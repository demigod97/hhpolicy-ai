### Phase 1: Create Citation Helper Functions (30 minutes)

**File**: `src/lib/citationHelpers.ts` (NEW FILE)

```typescript
import { supabase } from '@/integrations/supabase/client';

/**
 * Fetches the chunk content from the documents table based on citation data
 * @param sourceId - The source document ID
 * @param linesFrom - Starting line number of the chunk
 * @param linesTo - Ending line number of the chunk
 * @returns The text content of the chunk, or null if not found
 */
export async function fetchChunkContent(
  sourceId: string,
  linesFrom: number,
  linesTo: number
): Promise<string | null> {
  try {
    const { data, error } = await supabase
      .from('documents')
      .select('content, metadata')
      .eq('source_id', sourceId)
      .contains('metadata', {
        loc: {
          lines: {
            from: linesFrom,
            to: linesTo
          }
        }
      })
      .single();

    if (error) {
      console.error('Error fetching chunk content:', error);
      return null;
    }

    return data?.content || null;
  } catch (error) {
    console.error('Exception fetching chunk content:', error);
    return null;
  }
}
```

**What it does**:
- Queries `documents` table to fetch the actual text content of a citation
- Matches based on `source_id` and line range in metadata
- Returns raw markdown content for text search

```typescript
const openPDFViewer = async (
  documentId: string,
  pageNumber?: number,
  linesFrom?: number, // NEW
  linesTo?: number // NEW
) => {
  try {
    // Fetch document details
    const { data: source, error } = await supabase
      .from('sources')
      .select('id, title, pdf_file_path, pdf_storage_bucket')
      .eq('id', documentId)
      .single();

    if (error || !source) {
      toast({
        title: "Error",
        description: "Could not load document",
        variant: "destructive",
      });
      return;
    }

    // Fetch chunk content if line numbers provided (for highlighting) - NEW
    let highlightText: string | undefined;
    if (linesFrom !== undefined && linesTo !== undefined) {
      const chunkContent = await fetchChunkContent(documentId, linesFrom, linesTo);
      if (chunkContent) {
        highlightText = chunkContent;
      }
    }

    // Get signed URL for PDF
    const bucket = source.pdf_storage_bucket || 'sources';
    const { data: signedUrlData } = await supabase
      .storage
      .from(bucket)
      .createSignedUrl(source.pdf_file_path, 3600);

    if (signedUrlData?.signedUrl) {
      setSelectedDocument({
        id: source.id,
        title: source.title,
        fileUrl: signedUrlData.signedUrl,
        pageNumber,
        highlightText, // NEW
      });
      setShowPDFViewer(true);
    }
  } catch (error) {
    console.error('Error opening PDF:', error);
    toast({
      title: "Error",
      description: "Failed to open PDF viewer",
      variant: "destructive",
      });
  }
};
```

```typescript
// Function to search for text in PDF and navigate to it
const searchAndHighlight = async (textToFind: string) => {
  if (!pdfDocument || !textToFind) return;

  setIsSearching(true);
  try {
    // Clean the text - remove extra whitespace and markdown formatting
    const cleanText = textToFind
      .replace(/^##\s+/gm, '') // Remove markdown headers
      .replace(/\\\\/g, '') // Remove escaped backslashes
      .replace(/\n\n/g, ' ') // Replace double newlines with space
      .replace(/\s+/g, ' ') // Normalize whitespace
      .trim();

    // Extract a meaningful search phrase (first 50 chars)
    const searchPhrase = cleanText.substring(0, 50);

    // Search through all pages
    for (let pageNum = 1; pageNum <= pdfDocument.numPages; pageNum++) {
      const page = await pdfDocument.getPage(pageNum);
      const textContent = await page.getTextContent();
      const pageText = textContent.items
        .map((item: any) => item.str)
        .join(' ')
        .replace(/\s+/g, ' ');

      // Check if this page contains our search phrase
      if (pageText.includes(searchPhrase)) {
        console.log(`Found citation text on page ${pageNum}`);
        goToPage(pageNum);
        setSearchText(searchPhrase);
        break;
      }
    }
  } catch (error) {
    console.error('Error searching PDF:', error);
  } finally {
    setIsSearching(false);
  }
};
```

7. **Add custom text renderer** (line 129-150):

```typescript
// Custom text renderer to highlight search results
const customTextRenderer = React.useCallback((textItem: any) => {
  if (!searchText) return textItem.str;

  const searchLower = searchText.toLowerCase();
  const textLower = textItem.str.toLowerCase();

  if (textLower.includes(searchLower)) {
    // This text contains our search term - wrap it with mark tag
    const regex = new RegExp(`(${searchText})`, 'gi');
    const parts = textItem.str.split(regex);

    return parts.map((part: string, index: number) => {
      if (part.toLowerCase() === searchLower) {
        return `<mark key="${index}">${part}</mark>`;
      }
      return part;
    }).join('');
  }

  return textItem.str;
}, [searchText]);
```

3. **openPDFViewer()** calls `fetchChunkContent()` to get text from database

9. **customTextRenderer** wraps matching text with `<mark>` tags

## Technical Design Decisions

### Why Text-Based Search Instead of Line Numbers?

**Problem**: Citation data includes `chunk_lines_from` and `chunk_lines_to`, but these refer to line numbers in the **extracted markdown text**, not PDF page numbers.

**Solution**:
- Fetch the actual chunk text from the database
- Search for that text in the PDF using PDF.js text layer
- Highlight matched text regardless of page layout

**Benefits**:
- Works even if PDF layout differs from extracted text
- More reliable than trying to map line numbers to PDF positions
- Handles multi-column PDFs and complex layouts

### Why First 50 Characters?

**Reasoning**:
- Full chunk content can be 100+ words (too broad for highlighting)
- First 50 characters provide a unique, meaningful search phrase
- Balances specificity (finding exact location) with flexibility (handling minor formatting differences)

**Example**:
```
Full chunk: "## Liability & Compliance\n\n- Human Habitats is not responsible..."
Search phrase: "Liability & Compliance Human Habitats is not resp"
```

### Why Pulse Animation?

**User Experience**:
- Draws immediate attention to highlighted text when PDF opens
- Confirms that the highlight feature is working
- Subtle enough to not be distracting after initial appearance

## Known Limitations

### Limitation 1: First 50 Characters Only

**Issue**: Only the first 50 characters of the chunk are highlighted, not the entire chunk.

**Reason**: Highlighting large blocks of text (100+ words) reduces usefulness for pinpointing specific information.

**Workaround**: User can read full chunk context around highlighted text.

### Limitation 2: Exact Text Match Required

**Issue**: If PDF text differs from extracted markdown (e.g., OCR errors, formatting differences), highlighting may fail.

**Reason**: Text-based search requires exact (or very close) match.

**Workaround**: PDF still opens to best-guess page; user can search manually.

### Limitation 3: Single Page Display

**Issue**: If citation spans multiple pages, only first matching page is shown.

**Reason**: PDF.js renders one page at a time; highlighting across pages is complex.

**Future Enhancement**: Multi-page highlight support.

## Database Tables Used

### `documents` Table
```sql
SELECT
  id,
  source_id,
  content,
  metadata
FROM documents
WHERE source_id = 'citation_source_id'
  AND metadata @> '{"loc": {"lines": {"from": 1, "to": 25}}}';
```

### `sources` Table
```sql
SELECT
  id,
  title,
  pdf_file_path,
  pdf_storage_bucket
FROM sources
WHERE id = 'citation_source_id';
```

### `n8n_chat_histories` Table
```sql
SELECT
  message->'content'->'output'->0->'citations' as citations
FROM n8n_chat_histories
WHERE message->>'type' = 'ai';
```
