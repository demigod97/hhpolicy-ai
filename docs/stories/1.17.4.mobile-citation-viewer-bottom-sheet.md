# Story 1.17.4: Mobile Citation Viewer - Bottom Sheet Implementation

## Status
Ready for Review

## Story

**As a** PolicyAi user (All Roles),
**I want** to view citation sources when I click citation numbers on mobile devices,
**so that** I can verify AI responses with source documents and maintain trust in the system regardless of device.

## Context

**Priority:** Critical
**Epic:** 1.17 - Dashboard & Chat Enhancements
**UX Audit Reference:** Completed by Sally (UX Expert) - 2025-10-29

### Problem Statement

On **desktop**, users can click citation numbers in AI responses and the SourcesSidebar displays the relevant document excerpt in a dedicated right panel. This works seamlessly with the three-panel resizable layout.

On **mobile**, clicking citation numbers does **nothing visible** because the SourcesSidebar component is not rendered in the mobile layout. The `handleCitationClick` function executes and updates state, but there is no UI to display the citation content. This is a **complete feature failure** on mobile devices.

### Current System Integration

- **Existing Component**: `SourcesSidebar` (src/components/chat/SourcesSidebar.tsx) - Already functional on desktop
- **Existing State Management**: `selectedCitation` state and `handleCitationClick` handler in ChatInterface.tsx
- **Existing Pattern**: Sheet component used for PDF viewer modal (ChatInterface.tsx:293-308)
- **Technology**: React + TypeScript + shadcn/ui
- **Touch Points**: ChatInterface.tsx mobile layout (lines 266-289)

## Acceptance Criteria

### Functional Requirements

1. **Mobile Citation Display**
   - When a user clicks a citation number on mobile (<1100px viewport), a bottom sheet/drawer slides up from the bottom of the screen
   - The bottom sheet displays the same DocumentViewer content that desktop users see in the SourcesSidebar
   - Citation content includes highlighted text lines (linesFrom to linesTo) from the source document

2. **Bottom Sheet Behavior**
   - Sheet animates smoothly (300ms transition) sliding up from bottom
   - User can swipe down or tap X button to dismiss the sheet
   - Sheet height: 65vh (65% of viewport height) to preserve chat context
   - Sheet has semi-transparent backdrop (overlay)

3. **Mobile-Specific UX Enhancements**
   - Citation buttons are touch-optimized (minimum 44x44px touch target)
   - Sheet content is scrollable for longer document excerpts
   - Return focus to citation button when sheet is dismissed
   - Sheet works with or without ChatHistorySidebar visible

### Integration Requirements

4. Existing desktop SourcesSidebar behavior continues to work unchanged (three-panel layout preserved)
5. The `selectedCitation` state and `handleCitationClick` handler are reused without modification
6. Bottom sheet follows the same pattern as the existing PDF viewer Sheet (ChatInterface.tsx:293-308)
7. DocumentViewer component is reused in both desktop sidebar and mobile sheet

### Quality Requirements

8. Touch interactions follow iOS/Android native patterns (swipe to dismiss, smooth animations)
9. Component is covered by React Testing Library tests (mobile viewport simulation)
10. No regression in existing desktop citation functionality verified
11. Screen reader announces "Citation viewer opened" when sheet appears

### Existing Pattern Reference

The PDF viewer modal (ChatInterface.tsx:293-308) demonstrates the exact pattern to follow:

```tsx
<Sheet open={showPDFViewer} onOpenChange={setShowPDFViewer}>
  <SheetContent side="right" className="w-full sm:w-[80vw] sm:max-w-4xl p-0">
    <SheetHeader className="px-6 py-4 border-b">
      <SheetTitle>{selectedDocument?.title || 'Document Viewer'}</SheetTitle>
    </SheetHeader>
    {selectedDocument && (
      <div className="h-[calc(100vh-80px)]">
        <PDFViewer ... />
      </div>
    )}
  </SheetContent>
</Sheet>
```

**For mobile citation viewer, adapt as:**
```tsx
{/* Mobile Citation Bottom Sheet - Add after line 289 */}
{!isDesktop && (
  <Sheet
    open={selectedCitation !== null}
    onOpenChange={(open) => !open && setSelectedCitation(null)}
  >
    <SheetContent side="bottom" className="h-[65vh] p-0">
      <SheetHeader className="px-6 py-4 border-b">
        <SheetTitle>Citation Source</SheetTitle>
      </SheetHeader>
      <div className="h-[calc(65vh-64px)] overflow-hidden">
        <SourcesSidebar
          chatSessionId={sessionId}
          onDocumentSelect={(docId) => openPDFViewer(docId)}
          selectedCitation={selectedCitation}
          onCitationClose={() => setSelectedCitation(null)}
        />
      </div>
    </SheetContent>
  </Sheet>
)}
```

### Key Constraints

- **Mobile Breakpoint**: Use `isDesktop` hook (breakpoint: 1100px from useIsDesktop.tsx)
- **No State Changes**: Reuse existing `selectedCitation` state - do not create new state
- **Component Reuse**: SourcesSidebar must work in both desktop panel and mobile sheet
- **Touch Standards**: iOS/Android guideline of 44x44px minimum touch target
- **Animation Performance**: Must maintain 60fps during sheet transitions

### Technical Decisions

1. **Why Bottom Sheet over Modal?**
   - Preserves chat context (user can still see their conversation)
   - Familiar mobile pattern (iOS/Android native apps use bottom sheets)
   - Less disruptive than full-screen takeover
   - Easier to dismiss with swipe gesture

2. **Why 65vh Height?**
   - Shows substantial content without hiding entire chat
   - Allows user to see which citation they clicked
   - Provides enough space for typical document excerpts
   - Balances content visibility with context preservation

3. **Why Reuse SourcesSidebar?**
   - Maintains consistency between desktop and mobile
   - Reduces code duplication and maintenance burden
   - DocumentViewer component already handles citation highlighting
   - Ensures parity in citation viewing experience

**Mobile Testing Standards:**

```tsx
// Viewport setup for mobile tests
beforeEach(() => {
  window.matchMedia = jest.fn().mockImplementation(query => ({
    matches: query !== '(min-width: 1100px)', // Mobile viewport
    media: query,
    addEventListener: jest.fn(),
    removeEventListener: jest.fn()
  }));
});

// Test citation button touch target
test('citation buttons meet 44x44px minimum touch target', () => {
  const citationBtn = screen.getByRole('button', { name: /view citation/i });
  const { width, height } = citationBtn.getBoundingClientRect();
  expect(width).toBeGreaterThanOrEqual(44);
  expect(height).toBeGreaterThanOrEqual(44);
});

// Test bottom sheet opens on mobile
test('clicking citation on mobile opens bottom sheet', async () => {
  const user = userEvent.setup();
  render(<ChatInterface sessionId="test-123" />);

  const citationBtn = screen.getByRole('button', { name: '1' });
  await user.click(citationBtn);

  expect(screen.getByRole('dialog')).toBeInTheDocument();
  expect(screen.getByText('Citation Source')).toBeInTheDocument();
});

// Test sheet dismissal
test('sheet closes when user swipes down or clicks X', async () => {
  const user = userEvent.setup();
  render(<ChatInterface sessionId="test-123" />);

  // Open sheet
  await user.click(screen.getByRole('button', { name: '1' }));

  // Close via button
  await user.click(screen.getByRole('button', { name: /close/i }));

  expect(screen.queryByRole('dialog')).not.toBeInTheDocument();
});
```
