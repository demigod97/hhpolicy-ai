## Story

As a **System Administrator** uploading policy documents, I need the document processing pipeline to work correctly so that uploaded PDFs are automatically sent to N8N for text extraction and vector embedding, enabling AI-powered chat functionality.

### Current Problems

1. **Edge Function Not Called**: Documents are uploaded to storage and database, but the `process-document` edge function is never invoked
2. **Redirect to Wrong Port**: After upload, browser redirects to `http://localhost:8081/` instead of staying on dashboard
3. **No N8N Webhook Integration**: Documents never reach N8N for processing (text extraction, embedding generation)
4. **Processing Status Stuck**: Documents remain with `processing_status='completed'` but have no actual processed content

### Root Causes Identified

1. **Edge Function Call Commented Out**: In `useFileUpload.tsx` lines 112-132, the edge function call is commented out for testing
2. **Missing Environment Variables**: `DOCUMENT_PROCESSING_WEBHOOK_URL` not configured in Supabase
3. **Port Mismatch**: Some redirect or navigation is using port 8081 instead of 8082
4. **Real-time Subscriptions Work**: The recent real-time fixes are working, but no data is being processed

### Critical Bugs Discovered During Implementation

**Bug 1: White Screen Crash** 🐛
- **Location**: `src/components/document/DocumentUploader.tsx:57`
- **Issue**: `getFileIcon()` called `.startsWith()` on undefined `fileType`
- **Trigger**: Upload errors left `file.type` undefined, causing React crash
- **Fix**: Added null safety check: `if (!fileType) return File;`
- **Impact**: Prevented users from seeing upload errors

**Bug 2: Storage RLS Policies Missing** 🔒
- **Location**: `storage.objects` table
- **Issue**: RLS enabled but ZERO policies existed, blocking all uploads
- **Error**: `403 Unauthorized: new row violates row-level security policy`
- **Fix**: Created migration `20251020043506_fix_storage_policies_for_sources_bucket.sql` with 4 policies (INSERT, SELECT, UPDATE, DELETE)
- **Impact**: Complete upload failure for all users

**Bug 3: RLS Policy Violation on sources table** 🔐
- **Location**: `src/hooks/useFileUpload.tsx:49`
- **Issue**: Missing `uploaded_by_user_id` in INSERT statement
- **Error**: `403 Unauthorized: new row violates row-level security policy`
- **Fix**: Added `uploaded_by_user_id: user.id` to source creation
- **Impact**: Database INSERT failures for all uploads
