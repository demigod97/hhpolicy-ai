## Overview

Transform the minimal Help page into a comprehensive, user-friendly resource that documents the PolicyAi system, including templates, upload process, roles, access levels, and quick start guides. Remove unnecessary navigation elements and create an intuitive, searchable help center.

---

## Business Requirements

**As a** PolicyAi user (any role)
**I want** comprehensive, easy-to-find help documentation
**So that** I can quickly learn how to use the system without external support

**Value**:
- Reduced support requests to Company Operators
- Faster user onboarding for new staff
- Self-service learning for all features
- Clear role and permission documentation
- Improved user confidence and adoption

## Technical Requirements

### 1. Content Structure

**Required Sections** (6 major sections):

#### Section 1: Getting Started
- Welcome message
- Quick overview of PolicyAi purpose
- 5-minute quick start guide
- Common first tasks (upload document, search policies, download template)

#### Section 2: Templates
- What are templates?
- Template types (Policy, Process, Checklist)
- Access levels (General, Executive, Board)
- How to download templates
- How to use templates when uploading documents
- Link to Templates page

#### Section 3: Document Upload
- Step-by-step upload guide
- Metadata field explanations:
  - Title
  - Document category
  - Access level
  - Policy summary
  - Attributed to
  - Eligibility
  - Policy effective date
  - Policy last updated date
  - Policy information
  - Policy content
  - Notes, comments, executive notes
- File requirements (.doc, .docx, .pdf, 10MB limit)
- Processing time expectations
- Troubleshooting upload issues

#### Section 4: Roles & Permissions
- Role definitions:
  - **System Owner**: Developer/backend access, full system control
  - **Company Operator**: Document management, template upload, user role assignment
  - **General (Administrator)**: Staff access, view general-level documents
  - **Executive**: C-Level/VP access, view general + executive documents
  - **Board**: Board member access, view all documents
- Permission matrix table
- How to check your role
- How to request role changes (contact Company Operator)

#### Section 5: Access Levels & Document Security
- Document access level explanations
- Who can view which documents
- Role-based access control (RBAC) overview
- How access levels are assigned during upload
- Document visibility rules

#### Section 6: Frequently Asked Questions (FAQ)
- Why can't I see a document?
- How long does document processing take?
- Can I edit a document after upload?
- How do I change my role?
- What file types are supported?
- Why is my document flagged as outdated?
- How do I download a template?
- What's the difference between Notes and Comments?
- Who can see Executive Notes?
- How do I reset my password?

### 4. Content Examples

**Template Section Example**:
```markdown
## Templates

### What are Templates?

Templates are pre-formatted Word documents (.docx) designed to help you create
consistent, well-structured policy documents. PolicyAi provides 9 standardized
templates covering three document types and three access levels.

### Template Types

- **Policy**: Formal organizational policies (e.g., HR policies, IT policies)
- **Process**: Procedural documents describing how to complete tasks
- **Checklist**: Step-by-step checklists for compliance or operational tasks

### Access Levels

- **General**: Available to all staff (Administrators and above)
- **Executive**: Restricted to C-Level and VPs
- **Board**: Restricted to Board members only

### How to Download a Template

1. Click **Templates** in the navigation bar
2. Select the tab for your document type (Policy/Process/Checklist)
3. Choose the appropriate access level
4. Click **Download Template** on the desired template card
5. The Word document will download to your computer
6. Open in Microsoft Word and customize

[Go to Templates Page →](/templates)
```

**FAQ Section Example**:
```markdown
## Frequently Asked Questions

### Q: Why can't I see a document that my colleague can see?

**A**: Document visibility is controlled by access levels. If your colleague can see
a document you cannot, they likely have a higher role (e.g., Executive or Board)
and the document is restricted to that access level. Contact your Company Operator
if you believe you should have access.

### Q: How long does document processing take?

**A**: Most documents process within 2-5 minutes after upload. Larger documents
(5MB+) may take up to 10 minutes. You'll see a "Processing" status in the document
table. Once complete, the status changes to "Processed" and the document becomes
searchable in PolicyAi chat.

### Q: Can I edit a document after uploading it?

**A**: Currently, only Company Operators and System Owners can edit uploaded
documents. If you need to update a document, contact your Company Operator or
re-upload a corrected version.
```

### 5. Search Functionality (Optional Enhancement)

**Basic Search**:
```typescript
// Simple client-side search through section titles and content
function searchHelp(query: string) {
  const sections = getAllHelpSections();
  return sections.filter(section =>
    section.title.toLowerCase().includes(query.toLowerCase()) ||
    section.content.toLowerCase().includes(query.toLowerCase())
  );
}
```
